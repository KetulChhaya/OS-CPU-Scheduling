First Come First Serve (FCFS)



First Come First Serve(FCFS)

The process which will request for CPU first will be allocated the CPU first in this Algorithm. The implementation of FCFS is done by FIFO (First In First Out) Queue. When the CPU is free, it is allocated the process at head of the queue. So, when process enters ready queue, its PCB is linked onto tail of ready queue, And the running process is removed from the queue.
Characteristics of FCFS: 1)It supports non-preemptive and pre-emptive scheduling algorithm. 2)Jobs are always executed on a first-come, first-serve basis. 3)It is easy to implement and use. 4)This method is poor in performance, and the general wait time is quite high.

    Advantages:

  • Every process gets chance according to request they sent for use of CPU.
  • It is simple as there is no postponement of process while one process is running.

    • DisAdvantages:

  • As we know FCFS is non-preemptive, and also it doesnâ€™t allow the process which can give CPU efficient time management, it is not a good algorithm because it has long Average waiting time.
  • The reason we have Round-robin algorithm is to eliminate the disadvantages of FCFS algorithm as there is no mobility or freedom to make processes efficient.
