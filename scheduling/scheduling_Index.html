SOME BASIC TERMINOLOGIES OF CPU SCHEDULING

CPU Scheduling

CPU scheduling is basis of multiprogramming operating system. So in CPU scheduling we have several processes kept in a memory at the same time, to utilize this time productively. So during this time what happens is, when one process has to wait, the operating system takes CPU from that process and pass it to another process. This is what the CPU scheduler commits to do.

Why CPU Scheduling

The reason we need multiprogramming system is that in a simple computer system only one process runs at a time and all other processes have to wait for CPU to become free from its present process. So what happens is in simple computer system CPU remains idle when the system requires an I/O request. And this whole time as CPU remains idle this time is wasted in not utilizing the CPU. This is called time wasted. Dispatcher is a special program which comes into play after the scheduler. When the scheduler completes its job of selecting a process, it is the dispatcher which takes that process to the desired state/queue.

Goals of CPU Scheduling

  • Fare allocation of CPU
  • Maximum CPU utilization
  • Maximum throughput
  • Minimum turnaround time
  • Minimum waiting time
  • Minimum response time

CPU utilization

We want to keep CPU as busy as possible so that the process in the system remains productive.

Throughput

It is measure of the work or the number is processes completed by CPU per unit time. Throughout is more about CPU rather than process.

Turn Around Time

It is time taken between submission of a process to CPU and completion of process. It means it will include all the time period of CPU burst, I/O burst, ready state etc. Turn around time is more about process than about CPU.

Waiting Time

t is the time taken by the system when it is in ready state or ready queue. It needs to me minimized.

Response Time

Time taken by the process to give the 1st respose after submitting process to CPU.
